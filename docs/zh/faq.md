# 常见问题

## 这是机器人吗？

不是。MCP Telegram 作为 **userbot** 运行 — 使用您的个人 Telegram 账户，而不是机器人账户。这意味着它可以访问您所有的聊天、联系人和历史记录，就像手机上的 Telegram 应用一样。

## 安全吗？

安全。所有通信通过 MTProto 协议直接从您的设备发送到 Telegram 服务器。不经过任何第三方服务。会话文件以受限权限（0600）存储在本地。

## 账户会被封吗？

MCP Telegram 使用 Telegram 官方 API（MTProto），通过 GramJS 实现。只要不滥用（群发垃圾、自动刷屏、爬取数据），账户就不会有问题。请负责任地使用。

## 需要保持终端打开吗？

不需要。MCP Telegram 作为 MCP 客户端的一部分运行。客户端打开时服务器运行，关闭时停止。

## 可以使用多个账户吗？

可以！参见[多账户](/zh/guides/multiple-accounts)指南。

## 自建和云版本有什么区别？

| | 自建 | 云版本 ([mcp-telegram.com](https://mcp-telegram.com)) |
|---|---|---|
| **设置** | API 密钥、安装、QR 登录 | 只需扫码 |
| **运行位置** | 您的设备 | 我们的服务器 |
| **费用** | 免费（开源） | 免费版 + 付费版 |
| **客户端** | 任何 MCP 客户端 | Claude.ai、ChatGPT |
| **隐私** | 完全控制 | 消息在我们服务器上处理 |

## 支持频道吗？

支持。可以阅读频道消息、搜索频道。如果您是管理员，还可以发送消息和管理频道。

## 支持论坛群组（话题）吗？

支持。完整支持 — 列出话题、按话题阅读消息、发送到指定话题、按话题显示未读数。

## 有多少工具？

**59 个工具**（v1.24.0），涵盖消息、搜索、群组、贴纸、联系人、审核、账户管理等。查看[完整参考](/zh/tools/reference)。

## 可以不用 Node.js 吗？

可以！下载[预编译二进制文件](https://github.com/overpod/mcp-telegram/releases) — Linux、macOS 和 Windows 的独立可执行文件。
